# AI Brainstorm Canvas

AI Brainstorm Canvas - это коллаборативная онлайн-доска, где **2+ пользователя и AI-агент** участвуют в совместной brainstorming-сессии в одном пространстве.  
В отличие от обычного сценария, где AI открыт в отдельной вкладке и не видит, что происходит на доске, здесь агент **работает внутри canvas**: понимает контекст сессии, анализирует содержимое рабочего пространства и вносит вклад прямо на доске.

## Описание проекта

Во многих командах brainstorming проходит в Miro, FigJam или похожих инструментах параллельно с обсуждением в Zoom.  
При этом AI обычно существует отдельно - в виде чат-окна или стороннего инструмента. Из-за этого приходится:
- постоянно переключаться между окнами,
- заново объяснять контекст,
- вручную переносить результаты обратно на доску.

В этом проекте исследуется другой подход:

**AI - не внешний инструмент, а полноценный участник brainstorming-сессии.**

AI-агент:
- видит текущее состояние canvas,
- понимает контекст обсуждения,
- вносит вклад непосредственно в рабочее пространство,
- помогает не только текстом, но и пространственными действиями на доске.

## Основные возможности

### Совместная работа на canvas
- Общая доска для нескольких пользователей в реальном времени
- Возможность создавать, перемещать, редактировать и группировать идеи
- Поддержка совместной brainstorming-сессии в одном пространстве

### AI как участник сессии
- AI-агент работает **внутри canvas**
- Создает новые заметки и стикеры
- Помогает структурировать идеи
- Обновляет и развивает существующие элементы
- Связывает идеи между собой прямо на доске

### Коммуникация и контекст
- Пользователи могут взаимодействовать с системой через **текст и голос**
- AI учитывает не только запрос, но и:
  - текущее содержимое доски,
  - ход обсуждения,
  - уже созданные элементы,
  - общий контекст сессии

### Режимы работы AI-агента
Агент поддерживает несколько режимов участия:
- **generate** — генерация новых идей
- **cluster** — группировка и структурирование заметок
- **critic** — критика и проверка слабых идей
- **suggest_next** — предложение следующих шагов
- **visualize** — помощь с визуализацией идей
- **deepen** — развитие и углубление уже существующей идеи

### Пространственные действия агента
AI может выполнять действия непосредственно на canvas:
- создавать стикеры,
- создавать заголовки и секции,
- перемещать элементы,
- редактировать текст,
- распределять заметки по секциям,
- рисовать стрелки между связанными идеями.

Это позволяет воспринимать AI не как отдельный чат, а как реального участника совместной работы.

### Визуальное взаимодействие
- В системе поддерживается генерация визуального контента как части brainstorming-процесса
- Визуальные элементы появляются прямо на доске и могут использоваться наравне с текстовыми заметками
- AI может использовать визуальные элементы как часть своего вклада в сессию

## Примеры поведения AI-агента

Во время сессии AI может:
- предлагать новые идеи на основе обсуждения,
- объединять похожие заметки в группы,
- связывать идеи стрелками,
- развивать перспективные идеи через режим **deepen**,
- подсказывать следующие шаги, если команда застряла,
- помогать структурировать хаотичное brainstorming-пространство.

## Архитектура

Проект состоит из трех основных слоев:

1. **Canvas layer**  
   Общее визуальное пространство, где взаимодействуют пользователи и AI.

2. **Session / conversation layer**  
   Слой, в котором хранится история текстовых и голосовых взаимодействий в рамках сессии.

3. **Agent layer**  
   AI-слой, который анализирует состояние canvas и контекст обсуждения, после чего генерирует структурированные действия для выполнения прямо на доске.

## Локальный запуск

### 1. Клонируйте репозиторий
```bash
git clone <ссылка-на-репозиторий>
cd ai-canvas-agent
````

### 2. Установите зависимости

```bash
npm install
```

### 3. Добавьте переменные окружения

Создайте в корне проекта файл `.env.local` и укажите в нем необходимые ключи:

```env
OPENAI_API_KEY=your_openai_api_key
HIGGSFIELD_API_KEY=your_higgsfield_api_key
HIGGSFIELD_API_SECRET=your_higgsfield_api_secret
LIVEBLOCKS_SECRET_KEY=your_liveblocks_secret_key
NEXT_PUBLIC_LIVEBLOCKS_PUBLIC_KEY=your_liveblocks_public_key
```

### 4. Запустите проект

```bash
npm run dev
```

Приложение будет доступно по адресу:

```bash
http://localhost:3000
```

## Что нужно для работы проекта

Для запуска проекта необходимы:

* **OpenAI API key** — используется для логики AI-агента
* **Higgsfield API key и secret** — используются для генерации визуального контента
* **Liveblocks keys** — используются для совместной работы пользователей в реальном времени
